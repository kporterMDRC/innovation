# Document Data Dictionaries

## Ingredients

* Dataset information
* Variable descriptions
* Data types and formats
* Value ranges/codes
* Missing data codes
* Relationships/dependencies

## Instructions

Use this prompt template:

````
Please create a data dictionary for:

### Dataset Overview
* Name: [dataset name]
* Purpose: [main use]
* Collection: [how/when collected]
* Size: [rows x columns]
* Format: [file type/structure]

### Documentation needs:
* Detail level: [basic/intermediate/comprehensive]
* Special requirements:
    - [coding systems]
    - [unit standards]
    - [missing value codes]
    - [relationships]

### Variable categories:
* Identifiers: [list ID variables]
* Measures: [list measurement variables]
* Categories: [list categorical variables]
* Timestamps: [list temporal variables]
* Calculated: [list derived variables]

### Additional information:
* Quality notes: [known issues]
* Dependencies: [related datasets]
* Updates: [update frequency]
* Constraints: [data limitations]
````

## Tips

* Use consistent formatting
* Define all abbreviations
* Document missing values
* Include valid ranges
* Note data constraints
* Specify units clearly
* Document calculations
* Show relationships

## Serves

* Research datasets
* Survey data
* Time series
* Administrative data
* Merged datasets
* Derived datasets
* Database tables

## Example

````
Please create a data dictionary for:

### Dataset Overview
* Name: patient_outcomes
* Purpose: Track patient treatment outcomes
* Collection: Monthly hospital records
* Size: 10,000 x 15
* Format: CSV file, longitudinal

### Documentation needs:
* Detail level: comprehensive
* Special requirements:
    - ICD-10 diagnosis codes
    - LOINC lab test codes
    - Multiple missing value types
    - Patient-visit relationships

### Variable categories:
* Identifiers:
    - patient_id
    - visit_id
* Measures:
    - vital signs
    - lab results
* Categories:
    - diagnosis
    - treatment
* Timestamps:
    - admission_date
    - discharge_date
* Calculated:
    - length_of_stay
    - risk_score

### Additional information:
* Quality notes: Some legacy codes
* Dependencies: Links to pharmacy data
* Updates: Monthly
* Constraints: PHI restricted
````

## Standard Format

### 1. Metadata Section

```yaml
Dataset Information:
  name: [dataset name]
  version: [version number]
  date_created: [creation date]
  last_updated: [update date]
  owner: [data owner]
  contact: [contact information]
```

### 2. Variable Documentation

```yaml
Variables:
  [variable_name]:
    description: [clear description]
    type: [data type]
    format: [format specification]
    units: [if applicable]
    range: [valid range]
    codes: [value definitions]
    missing: [missing value codes]
    notes: [additional information]
    source: [data source]
    dependencies: [related variables]
```

### 3. Relationship Documentation

```yaml
Relationships:
  primary_keys: [key variables]
  foreign_keys:
    [variable]: [referenced dataset.variable]
  constraints:
    [constraint_name]: [constraint description]
```

## Content Guidelines

### 1. Variable Descriptions

* Clear and concise
* Avoid jargon
* Define terms
* Include context
* Note calculations
* Specify conditions

### 2. Data Values

* List valid values
* Define categories
* Specify units
* Note precision
* Document codes
* Explain missing values

### 3. Quality Information

* Known issues
* Data gaps
* Collection problems
* Quality checks
* Validation rules
* Update procedures

### 4. Usage Notes

* Common uses
* Limitations
* Precautions
* Best practices
* Related data
* Example queries
