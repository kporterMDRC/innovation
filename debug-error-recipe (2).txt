## Debugging Process Flow

### 1. Error Understanding
* Read error message carefully
* Note specific variables/functions
* Check error pattern
* Review related warnings

### 2. Problem Isolation
* Minimize code example
* Test components separately
* Verify data structures
* Check function inputs

### 3. Solution Approaches
* Fix common causes first
* Test assumptions
* Check documentation
* Search error message

### 4. Prevention Strategies
* Add error checking
* Improve documentation
* Create test cases
* Monitor dependencies

## Common Error Patterns

### 1. Data Problems
* Missing values
* Wrong data types
* Column name mismatches
* Empty datasets

### 2. Function Issues
* Wrong arguments
* Missing parameters
* Incompatible inputs
* Scope problems

### 3. Package Problems
* Version conflicts
* Missing dependencies
* Installation issues
* Load order problems

### 4. Memory/Performance
* Large datasets
* Resource limits
* Inefficient code
* System constraints
